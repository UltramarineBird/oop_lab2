Варианты 1, 3, 5…. Реализация «Магазина».
Цель: Изучение конструкторов для инициализации сложных объектов.
Задача: Создать класс, представляющий магазин с возможностью хранения продукции. Класс должен включать в себя:
Конструкторы: конструктор по умолчанию, перегруженный конструктор для добавления конкретной продукции к складу.
Деструктор: освобождает ресурсы магазина (например, закрытие соединений с базой данных).
Задачи продвинутого уровня: Внедрить управление памятью посредством smart-pointers для динамически создаваемых объектов; реализовать методы поиска и удаления товара в магазине.
